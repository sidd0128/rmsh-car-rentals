# Internationalization (i18n)

RMSH Rentals uses **i18next** and **react-i18next**, with all UI copy in `src/locales/en.json`. This matches the pattern used in Avy (`locales/en.json` + translation lookups) and prepares the app for additional languages later.

## Layout

| Path | Purpose |
|------|---------|
| `src/locales/en.json` | English strings (nested keys) |
| `src/core/i18n/index.ts` | i18n init + re-exports `useTranslation` |
| `src/types/i18n.d.ts` | TypeScript key autocomplete from `en.json` |

## Usage in components

```tsx
import { useTranslation } from '@core/i18n';

export const MyScreen = () => {
  const { t } = useTranslation();
  return <Text>{t('dashboard.title')}</Text>;
};
```

Interpolation:

```tsx
t('dashboard.returnsSoonDescription', { days: 3 });
```

## Navigation titles

Stack `options.title` uses `i18n.t` (no hooks in static options):

```tsx
import i18n from '@core/i18n';

options={{ title: i18n.t('navigation.fleet') }}
```

## Non-React helpers

```ts
import i18n from '@core/i18n';

export const formatPaymentStatusLabel = (status: PaymentStatus) =>
  i18n.t('common.received');
```

## Bootstrap

`import '@core/i18n'` runs in `App.tsx` before the UI tree. Jest loads it in `jest.setup.ts`.

## Adding a new language

1. Copy `src/locales/en.json` → `src/locales/sv.json` (or your locale).
2. Register in `src/core/i18n/index.ts`:

   ```ts
   import sv from '@locales/sv.json';

   resources: {
     en: { translation: en },
     sv: { translation: sv },
   },
   ```

3. Call `i18n.changeLanguage('sv')` from settings when you add a language picker.

## New strings

Add keys under the right section in `en.json` (e.g. `cars`, `customers`, `common`). Prefer reusing `common.*` for shared labels (Cancel, Save, Paid, etc.).
